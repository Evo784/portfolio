Depuis mes 14 ans je m'intéresse aux marchés financiers. Learning Lab est né de là : une plateforme full-stack en deux faces — Quant pour tester des stratégies de trading (backtest, walk-forward, optimisation), et Invest pour simuler la gestion d'une épargne (DCA, allocation multi-assets, Monte Carlo).

Le moteur de backtesting est entièrement maison — aucune dépendance à Backtrader ou vectorbt. Les indicateurs sont également implémentés en interne. Ce choix délibéré permet de montrer la maîtrise des outils plutôt que leur utilisation, et donne un contrôle total sur chaque calcul de la simulation.

  • ▸ core/ pur-Python — zéro import FastAPI ou UI. Testable en isolation totale.
  • ▸ Moteur vectorisé — signaux et frais précomputés en arrays NumPy, ~60 000× plus rapide qu'une boucle naïve. Une seule boucle autorisée : la machine d'états (flat/long/short).
  • ▸ Walk-forward — découpage en N fenêtres glissantes, ratio de robustesse OOS/IS pour détecter l'overfitting.
  • ▸ Grid search — jusqu'à 2 axes de paramètres, 500 combinaisons max, résultats en heatmap Sharpe/drawdown.
  • ▸ Data router — détecte automatiquement le provider (Binance pour BTCUSDT, yfinance pour SPY, MC.PA, GC=F…), cache Parquet keyed sur MD5.
Quant vs Invest

Même core.
Deux questions différentes.

La séparation n'est pas cosmétique : les deux faces posent des questions fondamentalement différentes, s'adressent à des profils différents, et appellent des interfaces différentes. Mais elles partagent le même moteur Python — aucune duplication de logique.

  • ▸ Face Quant — "Est-ce que ma stratégie fonctionne ?" Backtest + walk-forward + heatmaps, 16 stratégies pré-câblées (MA Cross, Supertrend, Ichimoku, RSI, Bollinger, VWAP…), données Binance.
  • ▸ Face Invest — "Comment gérer mon épargne ?" Simulateur DCA, constructeur d'allocation (60/40, 3-fund, crypto-heavy), projection Monte Carlo (Gaussienne + Bootstrap), multi-assets via yfinance.
  • ▸ Routes séparées : /invest/{allocation,dca,projection} vs /backtest, /walkforward, /optimize.
  • ▸ Pages Next.js séparées : app/invest/ vs app/quant/. Zéro feature creep entre les deux.

Chaque phase du projet a produit de nouveaux modules, et chaque module a ses tests avant de passer à la suivante. Ce n'est pas du happy-path : les cas limites de la machine d'états, les calculs de Sharpe annualisé, les indices IS/OOS exacts, la validation Pydantic côté API.

  • ▸ 17 tests — moteur backtest, indicateurs, métriques (Sharpe, Sortino, Calmar, max DD)
  • ▸ 37 tests — portfolio Invest : allocation, DCA, projection Monte Carlo
  • ▸ 19 tests — walk-forward : fenêtrage glissant, ratio de robustesse
  • ▸ 21 tests — grid search : combinatoires, garde-fou 500 combos, API routes
  • ▸ TypeScript strict côté frontend — zéro any sans justification explicite
