Pipeline de données
à 60 images par seconde.

Assetto Corsa expose ses données de simulation via une zone de mémoire partagée (shared memory). Ce projet lit ces données à 60Hz en temps réel — vitesse, RPM, G-forces, températures — les transmet via WebSocket, et les affiche dans un dashboard interactif avec moins de 15ms de latence bout en bout.

La motivation est simple : je joue à Assetto Corsa et je voulais voir mes données de conduite en live, comme un vrai pilote avec un ingénieur piste. Le challenge technique était de lire une API non documentée (shared memory layout), parser les données en C via ctypes, et les streamer sans perte vers le frontend.

  • ▸ Lecture shared memory via ctypes — pas de SDK, reverse engineering du layout de struct
  • ▸ FastAPI avec endpoint WebSocket dédié — un seul canal pour tous les clients
  • ▸ Buffer circulaire Python pour absorber les pics sans bloquer le thread de lecture
  • ▸ Reconnexion automatique côté client — le dashboard se reconnecte sans intervention
  • ▸ Pandas pour l'historique de session — analyses rétrospectives post-course
Données capturées

14 canaux
en temps réel.

Chaque donnée de la voiture mise à disposition par le jeu est capturée et exposée.

  • ▸ Vitesse (km/h) & RPM moteur
  • ▸ Rapport engagé & position pédale de gaz/frein
  • ▸ Angle de braquage (volant)
  • ▸ G-forces longitudinales, latérales, verticales
  • ▸ Températures des 4 pneumatiques
  • ▸ Niveau de carburant restant
  • ▸ Position sur la piste (coordonnées)
